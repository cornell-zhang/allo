# Catapult HLS PPA Analysis Flow

This document describes how to perform Power, Performance, and Area (PPA) analysis using the Catapult HLS backend in Allo, and how to further lower the design to ASIC using OpenROAD.

## Using the `ppa` Mode in Allo

Allo now supports a `ppa` mode for the Catapult target. This mode runs the full synthesis flow and extracts key metrics from the generated reports.

```python
import allo
from allo.ir.types import int32

def top(A: int32[32, 32], B: int32[32, 32]) -> int32[32, 32]:
    # ... your design ...
    pass

s = allo.customize(top)
# Customize for ASIC PPA
# s.partition(...)
# s.pipeline(...)

# Build with 'ppa' mode
mod = s.build(target="catapult", mode="ppa", project="my_ppa_project")

# Run synthesis and extract stats
stats = mod()
print(stats)
```

The `stats` dictionary contains:
- `Latency (cycles)`: Maximum latency reported by Catapult.
- `Area`: Total area reported (unit depends on the technology library).
- `Power`: Estimated power (if available in the report).

## ASIC PPA Breakdown

To achieve a breakdown into per-PE and interconnect costs, ensure that your Allo design uses hierarchical modules. Catapult HLS will preserve this hierarchy in the generated RTL unless fully flattened.

### Resource Analysis in Catapult

Catapult's `area.rpt` provides a hierarchical breakdown. You can find this in:
`<project_dir>/Catapult/<top_func>.v1/area.rpt`

### RTL-to-GDSII with OpenROAD

The synthesizable RTL generated by Catapult can be found in:
`<project_dir>/Catapult/<top_func>.v1/rtl.v` (or `.vhd`)

You can pass this RTL to the OpenROAD-flow-scripts for a full physical design PPA analysis:

1.  **Setup OpenROAD**: Follow the instructions at [TheOpenROADProject/OpenROAD-flow-scripts](https://github.com/TheOpenROADProject/OpenROAD-flow-scripts).
2.  **Configure for your design**: Create a `config.mk` pointing to the generated Verilog from Catapult.
3.  **Run Flow**: Execute `make` in the OpenROAD directory.
4.  **Extract Physical PPA**: Use OpenROAD's report generation to get accurate post-routing timing and power.

## Supported Technology Libraries
The Allo Catapult backend currently recognizes:
- `nangate-45nm_beh` (Default)
- `sky130`
